# 用户注册功能说明

## 功能特性

### 安全防护

- 图形验证码（4 位随机字符）
- 防暴力破解（错误 5 次失效）
- 验证码 5 分钟过期
- 用户名黑名单
- 密码强度验证

### 表单验证

- 用户名：3-20 字符，字母、数字、下划线
- 密码：至少 6 位
- 实时验证提示

### 默认用户组

- 支持为新注册用户自动分配默认用户组
- 在管理面板直接选择默认用户组
- 用户组需提前在“用户配置”中创建

## 启用条件

1. 存储类型必须是：`redis`、`upstash` 或 `kvrocks`
2. 使用站长账号打开 `/admin` -> `用户配置` -> `公开注册`
3. 开启“允许访客自行注册账号”

## 存储部署示例

### Docker 部署

```yaml
environment:
  - NEXT_PUBLIC_STORAGE_TYPE=redis
  - REDIS_URL=redis://redis:6379
```

### Vercel/ClawCloud

```bash
NEXT_PUBLIC_STORAGE_TYPE=upstash
UPSTASH_URL=<your_url>
UPSTASH_TOKEN=<your_token>
```

部署完成后，在管理面板中随时开启或关闭注册，无需修改环境变量或重新部署。

## 后台设置

| 设置                 | 说明                                       | 默认值 |
| -------------------- | ------------------------------------------ | ------ |
| 允许访客自行注册账号 | 立即控制登录页注册入口和服务端注册接口     | 关闭   |
| 新注册用户默认用户组 | 注册成功后自动加入该组，可限制其可用视频源 | 不分配 |

> **权限提示**：不分配用户组时，新注册用户默认可使用全部可用视频源。公网开放注册前，建议先创建受限用户组并设置为默认组。

## 旧版本兼容

已配置过 `NEXT_PUBLIC_ENABLE_REGISTRATION` 或 `DEFAULT_REGISTRATION_GROUP` 的旧部署，在升级后首次读取旧配置时会把环境变量值迁移为后台设置的初始值。此后以管理面板保存的设置为准；关闭注册不再需要重新部署。

## 使用流程

1. 访问 `/register` 页面
2. 填写用户名和密码
3. 输入图形验证码
4. 注册成功后自动跳转登录
5. 如配置了默认用户组，用户将自动加入该组

## 安全建议

- 建议用完即关闭注册功能
- 定期检查用户列表（`/admin` 页面）
- 及时封禁异常账号

## 常见问题

### 为什么显示"注册功能未开启"？

请让站长在 `/admin` 的“用户配置”中开启“允许访客自行注册账号”。

### 为什么显示"不支持注册"？

当前存储类型不支持多用户，需要部署 Redis/Upstash/Kvrocks。管理面板在 LocalStorage 模式下也会禁用注册开关。

### 注册后无法登录？

1. 检查数据库连接是否正常
2. 访问 `/admin` 查看用户是否创建成功
3. 确认用户未被封禁
